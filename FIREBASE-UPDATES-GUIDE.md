# دليل نشر التحديثات (طبقة Firebase)

## ✅ ما تم بناؤه بهذي الطبقة
- التطبيق الآن مرتبط فعلياً بمشروعك على Firebase (`halousha-daecb`).
- يستقبل إشعارات FCM صامتة، يحمّل الملف المحدّث بالخلفية، ويستخدمه فوراً بدون أي تثبيت APK جديد.
- يطلب إذن الإشعارات تلقائياً أول مرة يفتح (لأجل أي إشعار مرئي مستقبلي).
- مشترك تلقائياً بقناة (`all_devices`) — يعني أي رسالة ترسلها تصل لكل نسخة مثبتة من التطبيق (حالياً نسخة أخوك فقط، طبيعي).

## 🔑 خطوة لازمة الآن: ربط GitHub بصلاحية نشر الفنكشن

1. روح إلى Firebase Console → ⚙️ (إعدادات المشروع) → **Service accounts**.
2. اضغط **"Generate new private key"** → يحمّل ملف JSON.
3. روح لمستودعك بـ GitHub → **Settings** → **Secrets and variables** → **Actions**.
4. اضغط **"New repository secret"**:
   - **Name:** `FIREBASE_SERVICE_ACCOUNT`
   - **Value:** افتح ملف الـ JSON اللي حمّلته، انسخ **كل محتواه**، وألصقه هنا كامل.
5. اضغط **"Add secret"**.

بعد هذي الخطوة، أي رفع لمجلد `functions/` على GitHub ينشر الـ Cloud Function تلقائياً (تتابعه من تبويب Actions، نفس مبدأ بناء الـ APK).

## 📤 كيف تنشر تحديث فعلي لقسم (الآن، يدوياً بدون صفحة مسؤول)

1. **رفع الملف المحدّث لـ Firebase Storage:**
   - Firebase Console → Storage → Upload file
   - بعد الرفع، اضغط على الملف → "Copy download URL" (احفظ الرابط)

2. **تسجيل التحديث بـ Firestore:**
   - Firebase Console → Firestore Database
   - لو أول مرة: أنشئ مجموعة (Collection) اسمها بالضبط: `updates`
   - أضف وثيقة (Document) جديدة بداخلها، بهذي الحقول بالضبط:
     | الحقل | النوع | القيمة |
     |---|---|---|
     | `type` | string | `update` |
     | `relativePath` | string | مثلاً: `sections/cooking/index.html` |
     | `downloadUrl` | string | الرابط اللي نسخته بالخطوة 1 |

3. **خلصت.** Cloud Function تستلم الإشارة تلقائياً، ترسل إشعار صامت، وجهاز أخوك يحدّث الملف بالخلفية — كل هذا بدون أي تدخل إضافي منك.

> 💡 **مهم:** `relativePath` يجب أن يطابق **بالضبط** مسار الملف الأصلي داخل `assets/web/` بمشروع الأندرويد (نفس الحروف الكبيرة/الصغيرة ونفس المجلدات).

## 🔔 كيف ترسل إشعار مرئي (مستقبلاً، مثل تذكير)

نفس مبدأ التحديث، بس بمجموعة مختلفة:
- أنشئ مجموعة اسمها `notifications`
- أضف وثيقة فيها: `title` (مثلاً "تذكير") و `body` (نص الرسالة)
- يصل إشعار حقيقي يشوفه أخوك بشريط الإشعارات فوراً.

## ⚠️ ملاحظات مهمة
- **أول إشعار صامت ما يوصل إلا بعد ما يفتح أخوك التطبيق مرة واحدة بعد التثبيت** (هذا يسجّله بقناة `all_devices`).
- لو الجهاز يفعّل توفير بطارية قوي، ممكن يتأخر استلام الإشعار الصامت — التطبيق يتحقق أيضاً من التحديثات لحظياً وقت ما يكون مفتوح (عبر `sections-loader.js`)، فهذا يعالج جزء من هذا التأخير لأقسام الـ hub، لكن تحديثات الملفات الداخلية (مثل صورة بقسم الطبخ) تعتمد فعلياً على وصول الإشعار.
- الرابط (`downloadUrl`) من Firebase Storage يبقى صالح دائماً إلا لو حذفت الملف أو غيّرت صلاحيات الوصول.
