# دليل الاستخدام — الواجهة الرئيسية الموحّدة

## الفكرة باختصار
هذا الهيكل **لا يلمس أقسامك الحالية إطلاقاً**. كل قسم يبقى ملف/مجلد منفصل تماماً
بكوده وتصميمه وقواعد بياناته زي ما هو. الواجهة الرئيسية (`index.html`) فقط
"تستدعي" كل قسم داخل `iframe` عند الضغط على بطاقته — فما فيه أي تعارض أكواد.

---

## كيف تدمج ملفاتك الحقيقية

كل مجلد داخل `sections/` فيه حالياً ملف `index.html` مؤقت (placeholder).

1. روح لمجلد القسم، مثلاً `sections/music/`
2. احذف الملف المؤقت
3. انسخ **كل ملفات قسم الأغاني الحقيقية** (html, js, css, صور...) داخل هذا المجلد
4. تأكد إن نقطة الدخول لازم تكون اسمها بالضبط `index.html`
   (لو ملفك الرئيسي اسمه مختلف، إما تسميه `index.html` أو تعدّل المسار
   في `data-src` داخل `index.html` الرئيسي وفي `service-worker.js`)
5. كرر نفس الخطوات لباقي الأقسام: drawing, ai, stories, cooking, photos, games, admin

> ⚠️ لو أي قسم يستخدم مسارات نسبية لملفاته الداخلية (`./style.css`, `./img/x.png`)
> فهذي تشتغل تلقائياً طالما كل شي داخل مجلد القسم نفسه — لا تغيّرها.

---

## الأيقونات (مطلوبة لتثبيت التطبيق على أندرويد)
ضع صورتين داخل `assets/icons/`:
- `icon-192.png` (192×192 بكسل)
- `icon-512.png` (512×512 بكسل)

بدونها التطبيق يشتغل عادي بالمتصفح، لكن خاصية "إضافة للشاشة الرئيسية"
(تثبيت كتطبيق حقيقي) ما تكتمل بشكل صحيح.

---

## ربط Firebase الفعلي (تحديثات المسؤول)
افتح `assets/js/update-checker.js` ودوّر على هذا الجزء:

```js
async function fetchRemoteVersionPlaceholder() {
  return null; // ضع هنا الاستدعاء الفعلي لـ Firebase
}
```

استبدله باستدعاء حقيقي، مثال (Realtime Database):

```js
import { getDatabase, ref, get } from "https://www.gstatic.com/firebasejs/10.x.x/firebase-database.js";

async function fetchRemoteVersionPlaceholder() {
  const db = getDatabase();
  const snap = await get(ref(db, 'app_meta/last_update'));
  return snap.exists() ? snap.val() : null;
}
```

وفي تطبيق المسؤول، عند أي عملية حفظ/تحديث، اكتب timestamp جديد في
`app_meta/last_update` — وهذا كافي ليكتشف التطبيق وجود تحديث ويحدّث
القسم المفتوح تلقائياً.

---

## عند تحديث كود/تصميم التطبيق نفسه (مو بيانات Firebase)
افتح `service-worker.js` وغيّر السطر:

```js
const CACHE_VERSION = 'v1';
```

إلى `'v2'` (أو أي رقم جديد) في كل مرة ترفع نسخة جديدة من الملفات.
هذا يجبر كل نسخة قديمة محفوظة على الهاتف تنحذف ويتحمّل كل شي من جديد —
يصير تلقائياً بدون أي تدخل من المستخدم.

---

## النقل بين الهواتف
بما إن التطبيق Offline-First مرتبط بـ Firebase:
1. افتح التطبيق على الهاتف الجديد مع وجود إنترنت
2. خلّه يشتغل مرة وحدة (يسحب آخر نسخة من Firebase + يحفظها محلياً)
3. بعدها يشتغل أوفلاين عادي بدون ما يحتاج إنترنت كل مرة

---

## ملخص بنية الملفات

```
index.html              ← الواجهة الرئيسية (الهب) — لا تلمسها إلا لتعديل التصميم العام
manifest.json            ← إعدادات تثبيت PWA
service-worker.js         ← التشغيل الأوفلاين + التحديث التلقائي
assets/
  css/shell.css            ← تصميم الواجهة الرئيسية فقط
  js/shell.js               ← منطق التنقل بين الأقسام
  js/update-checker.js      ← التحقق من التحديثات (Firebase + كود)
  icons/                    ← أيقونات PWA (تحتاج تضيفها)
sections/
  music/      drawing/      ai/        stories/
  cooking/    photos/       games/     admin/
  ← كل مجلد = قسمك الحقيقي بالكامل بدون أي تعديل
```
